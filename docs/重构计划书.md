## 1. 项目概述

### 1.1 项目背景

> ⚠️ **免责声明**：PCL-CE.Neo 是基于 [PCL Community Edition](https://github.com/PCL-Community/PCL-CE) 的社区驱动跨平台重构项目。PCL Community Edition 的原作者为社区贡献者。本项目仅为社区开发者发起的开源项目，旨在为非 Windows 用户提供 PCL 启动器的使用体验。

PCL Community Edition (PCL CE) 是一款广受欢迎的 Minecraft 启动器，目前仅支持 Windows 平台。随着社区的发展，越来越多的用户希望在 macOS 和 Linux 上使用 PCL CE。PCL-CE.Neo 项目旨在将 PCL CE 重构为基于 Uno Platform 的跨平台应用，同时保留所有现有功能。

### 1.2 项目目标

- **跨平台支持**：支持 Windows、macOS、Linux 三大桌面平台
- **架构升级**：从 .NET 8 + WPF 升级到 .NET 10 + Uno Platform
- **功能完整性**：保留 100% 的现有功能
- **性能相当**：跨平台版本性能不低于现有 Windows 版本
- **可维护性**：提高代码质量和可维护性

### 1.3 关键成功因素

1. 清晰的架构抽象层
2. 充分的自动化测试
3. 渐进式迁移策略
4. 活跃的社区参与

## 2. 目标与范围

### 2.1 功能范围

#### 必须保留的功能
- ✅ Minecraft 版本管理和安装
- ✅ Mod 管理（CurseForge、Modrinth 集成）
- ✅ 帐户管理和认证
- ✅ 游戏启动和日志查看
- ✅ 主题和自定义 UI
- ✅ 多人游戏联机功能

#### 新增加的功能
- 🌐 跨平台配置同步
- 🎨 各平台原生体验优化
- 📦 各平台原生打包

### 2.2 平台范围

| 平台 | 优先级 | 目标版本 | 说明 |
|------|--------|----------|------|
| Windows 10/11 | P0 | 首发 | 与现有功能一致 |
| macOS 12+ | P0 | 首发 | 完整支持 |
| Linux (Ubuntu 22.04+) | P1 | 首发 | 完整支持 |

- 不包含移动端（iOS/Android）支持（可后续扩展）

### 4.2 架构层次图

```
┌─────────────────────────────────────────────────────────┐
│                   应用层 (PCL-CE.Neo.App)               │
│  ┌───────────────────────────────────────────────────┐ │
│  │           Uno Platform UI (PCL-CE.Neo.UI)          │ │
│  │  Pages • Controls • Themes • Animations           │ │
│  └───────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│                  服务层 (Services)                      │
│  ┌─────────────────┐  ┌─────────────────────────────┐ │
│  │   核心业务逻辑   │  │  平台抽象层 (Abstractions)  │ │
│  │  (PCL-CE.Neo.Core)  │  │ IPlatformService           │ │
│  │                  │  │ IWindowService              │ │
│  │  配置 • 网络     │  │ IJavaScanner                │ │
│  │  日志 • 任务     │  │ IAudioService               │ │
│  │  Minecraft 逻辑  │  │ IThemeService               │ │
│  └─────────────────┘  └─────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│              平台实现层 (Platform Implementations)      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ Windows  │  │  macOS   │  │  Linux   │             │
│  └──────────┘  └──────────┘  └──────────┘             │
└─────────────────────────────────────────────────────────┘
```

### 4.3 关键设计原则

1. **依赖倒置原则**：高层模块不依赖低层模块，都依赖抽象
2. **接口隔离**：定义清晰的平台抽象接口
3. **渐进式迁移**：保持现有版本可工作，逐步迁移
4. **可测试性**：所有组件都应易于单元测试
5. **向后兼容**：配置文件、数据格式保持兼容

namespace PCL.Core.Abstractions;

public interface IPlatformService
{
    string PlatformName { get; }

    string Architecture { get; }

    string GetLocalApplicationDataPath();

#### Windows 实现示例
```csharp
// 文件：src/PCL.Platform/Windows/WindowsPlatformService.cs
using PCL.Core.Abstractions;

namespace PCL.Platform.Windows;

public class WindowsPlatformService : IPlatformService
{
    public string PlatformName => "Windows";
    public string OSVersion => Environment.OSVersion.VersionString;
    public string Architecture => RuntimeInformation.ProcessArchitecture.ToString();
    
    public void OpenUrl(string url)
    {
        Process.Start(new ProcessStartInfo(url) { UseShellExecute = true });
    }
    
    public void OpenFolder(string path)
    {
        Process.Start("explorer.exe", path);
    }
    
    public string GetLocalApplicationDataPath()
    {
        return Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData);
    }
    
    public string GetTempPath() => Path.GetTempPath();
}
```

## 8. 质量保证计划

### 8.1 强制代码质量规则（🚨 强制执行）

> ⚠️ **【死命令】严禁简化代码，严禁空壳实现**
> 
> 以下规则已强制执行，无任何例外。

#### 8.1.1 禁止的代码模式

| 违规代码 | 说明 | 替代方案 |
|----------|------|----------|
| `Console.WriteLine` | 禁止使用控制台输出 | 使用日志系统（ILogger） |
| `// TODO` | 禁止TODO注释 | 必须完成实现或创建Issue |
| `throw new NotImplementedException()` | 禁止空实现抛出未实现异常 | 必须完整实现功能 |
| 空方法体 | 禁止空壳实现 | 必须有完整实现 |

#### 8.1.2 验收标准

**每次代码提交前必须验证以下内容**：

```bash
# 验证命令 - 所有结果必须为0
grep -r "Console\.WriteLine" PCL-CE.Neo*/**/*.cs  # 必须为0
grep -r "NotImplementedException" PCL-CE.Neo*/**/*.cs  # 必须为0（除非已批准例外）
grep -r "// TODO" PCL-CE.Neo*/**/*.cs  # 必须为0
grep -r "throw new$" PCL-CE.Neo*/**/*.cs  # 只允许参数校验和已批准的例外
```

#### 8.1.3 唯一例外情况

如果某个功能在当前平台技术上无法实现，必须：

1. **先查官方文档确认**真的无法实现
2. **把错误信息和搜索结果**贴给项目负责人确认
3. **获得批准后**，才能使用：
   ```csharp
   throw new PlatformNotSupportedException("具体原因：XXX平台不支持YYY API，官方文档链接：...");
   ```
4. **同时更新 PLATFORM_TODO.md**，在对应功能处标注：
   ```markdown
   - [ ] `XXX功能` - ❌ 平台不支持：具体原因（已申请例外）
   ```

#### 8.1.4 平台实现要求

- 每一个接口方法必须有**完整的、真实的实现**
- 每一个平台（Windows/macOS/Linux）的实现必须**真正调用该平台的 API**
- 不允许"先写个空壳，以后再说"

#### 8.1.5 违规处理

违反以上规则的代码：
- **不允许合并**到主分支
- **必须修复**后才能继续开发
- 多次违规将触发代码审查强化要求
