# 贡献指南

欢迎为 PCL-CE.Neo 项目做出贡献！

## 如何开始

1. 阅读 [架构设计文档](../architecture/ARCHITECTURE.md)
2. 了解 [项目结构](../architecture/PROJECT_STRUCTURE.md)
3. 配置 [开发环境](DEVELOPMENT.md)
4. 阅读 [代码规范](CODING_STANDARDS.md)

## 贡献类型

### 报告 Bug

在提交 Issue 之前：
- 搜索现有 Issue 确认没有重复
- 提供详细的复现步骤
- 说明你的环境（操作系统、.NET 版本等）
- 提供错误日志和截图（如果适用）

### 提交功能请求

- 清晰描述功能需求
- 说明解决的问题
- 如果可能，提供实现建议

### 代码贡献

1. Fork 项目仓库
2. 创建功能分支
3. 实现更改
4. 确保测试通过
5. 提交 Pull Request

## 开发指南

### 开发环境设置

参考 [开发环境配置](DEVELOPMENT.md) 设置你的开发环境。

### 代码风格

严格遵循 [代码规范](CODING_STANDARDS.md)。

### 测试

- 新功能必须包含单元测试
- 修复 bug 时添加回归测试
- 确保所有测试通过后再提交 PR

### 文档

- 更新与你更改相关的文档
- 如果添加新功能，添加相应文档
- 更新代码注释和 XML 文档

## Pull Request 流程

1. 创建一个描述性的 PR 标题
2. 填写 PR 模板中的所有字段
3. 确保 CI 检查通过
4. 等待代码审查
5. 根据反馈进行修改
6. PR 被合并后，删除你的分支

## 行为准则

- 尊重所有贡献者
- 提供建设性的反馈
- 接受并给予帮助
- 关注项目目标

## 获取帮助

有问题？可以：
- 查看项目文档
- 搜索 Issues 和 Discussions
- 提交新的 Issue

## 许可证

通过贡献代码，你同意你的贡献将在项目的现有许可证下发布。
