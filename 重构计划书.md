## 1. 项目概述

### 1.1 项目背景

PCL Community Edition (PCL CE) 是一款广受欢迎的 Minecraft 启动器，目前仅支持 Windows 平台。随着社区的发展，越来越多的用户希望在 macOS 和 Linux 上使用 PCL CE。本项目旨在将 PCL CE 重构为基于 Uno Platform 的跨平台应用，同时保留所有现有功能。

### 1.2 项目目标

- **跨平台支持**：支持 Windows、macOS、Linux 三大桌面平台
- **架构升级**：从 .NET 8 + WPF 升级到 .NET 10 + Uno Platform
- **功能完整性**：保留 100% 的现有功能
- **性能相当**：跨平台版本性能不低于现有 Windows 版本
- **可维护性**：提高代码质量和可维护性

### 1.3 关键成功因素

1. 清晰的架构抽象层
2. 充分的自动化测试
3. 渐进式迁移策略
4. 活跃的社区参与

## 2. 目标与范围

### 2.1 功能范围

#### 必须保留的功能
- ✅ Minecraft 版本管理和安装
- ✅ Mod 管理（CurseForge、Modrinth 集成）
- ✅ 帐户管理和认证
- ✅ 游戏启动和日志查看
- ✅ 主题和自定义 UI
- ✅ 多人游戏联机功能

#### 新增加的功能
- 🌐 跨平台配置同步
- 🎨 各平台原生体验优化
- 📦 各平台原生打包

### 2.2 平台范围

| 平台 | 优先级 | 目标版本 | 说明 |
|------|--------|----------|------|
| Windows 10/11 | P0 | 首发 | 与现有功能一致 |
| macOS 12+ | P0 | 首发 | 完整支持 |
| Linux (Ubuntu 22.04+) | P1 | 首发 | 完整支持 |
| Web (WASM) | P2 | 后续 | 功能受限版本 |

### 2.3 排除范围

- 不包含移动端（iOS/Android）支持（可后续扩展）
- 不重写核心业务逻辑，仅做适配
- 不改变现有 API 接口（保持向后兼容）

### 4.2 架构层次图

```
┌─────────────────────────────────────────────────────────┐
│                    应用层 (App)                         │
│  ┌───────────────────────────────────────────────────┐ │
│  │              Uno Platform UI (PCL.UI)             │ │
│  │  Pages • Controls • Themes • Animations          │ │
│  └───────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│                  服务层 (Services)                      │
│  ┌─────────────────┐  ┌─────────────────────────────┐ │
│  │   核心业务逻辑  │  │     平台抽象层 (Abstractions)│ │
│  │   (PCL.Core)    │  │  IPlatformService           │ │
│  │                 │  │  IWindowService             │ │
│  │  配置 • 网络    │  │  IJavaScanner               │ │
│  │  日志 • 任务    │  │  IAudioService              │ │
│  │  Minecraft 逻辑 │  │  IThemeService              │ │
│  └─────────────────┘  └─────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│                平台实现层 (Platform Implementations)    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │ Windows  │  │  macOS   │  │  Linux   │            │
│  └──────────┘  └──────────┘  └──────────┘            │
└─────────────────────────────────────────────────────────┘
```

### 4.3 关键设计原则

1. **依赖倒置原则**：高层模块不依赖低层模块，都依赖抽象
2. **接口隔离**：定义清晰的平台抽象接口
3. **渐进式迁移**：保持现有版本可工作，逐步迁移
4. **可测试性**：所有组件都应易于单元测试
5. **向后兼容**：配置文件、数据格式保持兼容

## 6. 技术实施细节

### 6.1 平台抽象示例

#### IPlatformService 接口定义
```csharp
// 文件：src/PCL.Core.Abstractions/IPlatformService.cs
namespace PCL.Core.Abstractions;

public interface IPlatformService
{
    string PlatformName { get; }
    string OSVersion { get; }
    string Architecture { get; }
    
    void OpenUrl(string url);
    void OpenFolder(string path);
    string GetLocalApplicationDataPath();
    string GetTempPath();
}
```

#### Windows 实现示例
```csharp
// 文件：src/PCL.Platform/Windows/WindowsPlatformService.cs
using PCL.Core.Abstractions;

namespace PCL.Platform.Windows;

public class WindowsPlatformService : IPlatformService
{
    public string PlatformName => "Windows";
    public string OSVersion => Environment.OSVersion.VersionString;
    public string Architecture => RuntimeInformation.ProcessArchitecture.ToString();
    
    public void OpenUrl(string url)
    {
        Process.Start(new ProcessStartInfo(url) { UseShellExecute = true });
    }
    
    public void OpenFolder(string path)
    {
        Process.Start("explorer.exe", path);
    }
    
    public string GetLocalApplicationDataPath()
    {
        return Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData);
    }
    
    public string GetTempPath() => Path.GetTempPath();
}
```

### 6.2 依赖注入配置

```csharp
// 文件：src/PCL.App/Platforms/Windows/WindowsProgram.cs
using Microsoft.Extensions.DependencyInjection;
using PCL.Core.Abstractions;
using PCL.Platform.Windows;

namespace PCL.App.Windows;

public static class WindowsServiceCollectionExtensions
{
    public static IServiceCollection AddWindowsPlatformServices(this IServiceCollection services)
    {
        services.AddSingleton<IPlatformService, WindowsPlatformService>();
        services.AddSingleton<IWindowService, WindowsWindowService>();
        services.AddSingleton<IJavaScanner, RegistryJavaScanner>();
        services.AddSingleton<IAudioService, WindowsAudioService>();
        return services;
    }
}
```
